# Задача 3n + 1

В 1932 году немецкий математик Лотар Коллатц предложил следующую простую задачу, которая до сих пор остается нерешенной.

Рассмотрим следующий алгоритм:

1. ввести $n$;
2. вывести $n$;
3. если $n = 1$, то остановиться;
4. если $n$ нечетное,
   4.1. то $n = 3 * n + 1$;
   4.2. иначе $n = n / 2$;
5. перейти на 2;

При $n = 22$ будет напечатана следующая последовательность чисел: $22, 11, 34, 17, 52, 26, 13, 40, 20, 10, 5, 16, 8, 4, 2, 1$.

По утверждению Коллатца приведенный алгоритм всегда остановится для любого значения $n$, хотя это утверждение до сих пор не доказано.

Для заданного $n$ можно определить количество чисел, которое будет напечатано алгоритмом. Это количество называется длиной цикла. Например, для  $n = 22$ длина цикла равна 16.

В задаче следует найти максимальное значение среди всех длин циклов чисел от $i$ до $j$ включительно.

## входные данные

В первой строке содержится пара целых чисел $i$ и $j$, разделенных пробелом.

## выходные данные

Вывести число из промежутка от $i$ до $j$ (включительно) с наибольшей длиной цикла, разделенное пробелом с длиной цикла этого числа. Если таких чисел несколько, вывести наименьшее из них.

## ограничения

* $0 \leq i, j \leq 10000$
* time limit: 1s
* memory limit: 8m

## пример

| INPUT | OUTPUT |
| ----- | ------ |
| 1 15  | 9 20   |
